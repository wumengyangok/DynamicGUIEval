Role: 你是一位资深的手机产品经理和用户体验（UX）专家，擅长设计基于情境感知的智能推荐系统。

Task: 请根据我提供的“场景模版”和“目标场景”，生成一套完整的用户行为路径数据。你需要模拟用户在特定场景下，手机系统如何根据时间、环境和用户习惯动态推送App卡片。

Input Modules:
Module 1: 场景模版（参考用）
场景：从上海到北京听演唱会。
流程：Step1(用户画像) -> Step2(起飞前一天) -> Step3(航班信息注入) -> Step4(起飞前2.5h) -> Step5(到达北京) -> Step6(演唱会结束)。
Module 2: 目标场景（需执行）
场景：用户需要去上海出差。
流程：请你根据这一核心意图，合理拆解出至少3个关键时间节点（Step）。其中，Step1放用户画像信息，即用户平时的习惯。

Requirements:
数据结构: 输出必须严格为 .json 格式（数组形式 [{{step1}, {step2}, ...}]）。
字段定义: 每个Step包含以下键值（Step1只需要有step_id和context_injection的内容，其他都设置为null）： 
step_id: 步骤编号。
context_injection: 注入的环境信息（如时间、地点、事件）。
main_app: 推送的主App名称。
sub_app: 推送的副App名称。
relevance_level: 主/副App相关性（分为：1. 非常相关；2. 中等相关；3. 仅有某些信息相关），并简述理由。
main_card_thumb_url: "main_thumb.png" (占位符)。
main_card_detail_url: "main_detail.png" (占位符)。
sub_card_thumb_url: "sub_thumb.png" (占位符)。
sub_card_detail_url: "sub_detail.png" (占位符)。
main_app_detail_prompt: 用于文生图模型生成主App详情图的中文提示词 。
sub_app_detail_prompt: 用于文生图模型生成副App详情图的中文提示词。
main_app_thumb_prompt:用于文生图模型生成主App缩略图的中文提示词 。
sub_app_thumb_prompt:用于文生图模型生成副App缩略图的中文提示词。

生成逻辑: 
1) 动态推送: 随着Step推进，主/副App应根据用户意图动态调整。
2) Prompt设计: 提示词需包含“手机App UI设计风格”、“无背景”、“高清晰度文本”等描述，并要求图片生成模型生成四张图，即相关的主/副app缩略图 （正方形/长宽大致相等的无规则图形、文字清晰）和主/副app详情图 （**实际手机app GUI截图, 内容较为详细**） ，且要求**图片里面的文本为中文，文本高度清晰，逻辑正确且可读**。同时，确保每个step中main_app_detail_prompt、 sub_app_detail_prompt、 main_app_thumb_prompt和sub_app_thumb_prompt生成逻辑符合多轮对话逻辑，以main_app_detail_prompt作为第一轮，并确保后续图片在UI风格上具有连贯性，但主app和副app的缩略图的视觉上要有差别。
